Support Vector Machine (SVM)

SVM adalah algoritma klasifikasi yang bekerja dengan mencari hyperplane terbaik untuk memisahkan data dari kelas yang berbeda. Kalau datanya dua dimensi, hyperplane bisa dibayangkan sebagai sebuah garis. Kalau dimensinya lebih tinggi, bentuknya menjadi bidang atau pemisah yang lebih umum.

Tujuan utama SVM bukan hanya memisahkan data, tetapi mencari pemisah yang memiliki margin paling besar. Margin adalah jarak antara batas keputusan dengan titik data terdekat dari masing-masing kelas. Titik-titik yang paling dekat dengan batas ini disebut support vectors.

Penjelasan sangat sederhana

Bayangkan ada dua kelompok titik di papan. Tugas SVM adalah menarik garis pemisah di antara dua kelompok itu.

Tetapi SVM tidak asal menarik garis. Ia mencari garis yang memberi jarak aman paling lebar dari kedua kelompok. Jadi, garis yang dipilih benar-benar dianggap paling aman untuk memisahkan dua kelas.

Kenapa margin penting?

Karena margin yang lebar biasanya membuat model lebih baik saat menghadapi data baru. Secara intuitif, model tidak terlalu mepet ke salah satu kelas, sehingga pemisahnya lebih aman.

Kernel

Kernel menentukan bentuk pemisah. Linear cocok untuk pemisahan garis lurus. Poly dan RBF membantu saat pola data lebih melengkung.

Gamma

Gamma mengatur seberapa jauh pengaruh satu titik data. Nilai besar membuat batas lebih sensitif terhadap pola di sekitar titik.

Regularization (C)

C mengatur seberapa ketat model terhadap kesalahan klasifikasi di data latihan. C besar cenderung lebih ketat.

Degree

Degree dipakai jika kernel polynomial digunakan. Nilainya memengaruhi tingkat kelengkungan atau kompleksitas pemisah.

Kelebihan SVM

  • Mencari margin maksimal sehingga sering punya kemampuan generalisasi yang baik.
  • Bisa menangani data non-linier dengan bantuan kernel.
  • Sering bekerja baik pada dataset yang tidak terlalu besar.
  • Efektif pada data berdimensi tinggi.

Kekurangan SVM

  • Bisa lambat untuk dataset besar.
  • Parameter seperti C, gamma, dan kernel perlu dipilih dengan hati-hati.
  • Tidak selalu mudah dijelaskan secara intuitif, terutama jika memakai kernel non-linier.
  • Probabilitas kelas tidak muncul secara langsung tanpa pengaturan tambahan.

Decision Tree

Decision Tree adalah algoritma yang membuat keputusan secara bertahap seperti pohon. Model ini membagi data berdasarkan pertanyaan-pertanyaan sederhana. Karena bentuknya bercabang, algoritma ini sering menjadi salah satu yang paling mudah dijelaskan kepada pemula.

Struktur dasarnya terdiri dari tiga bagian utama: root node sebagai titik awal, decision node sebagai tempat munculnya pertanyaan, dan leaf node sebagai hasil akhir keputusan.

Root node

Ini adalah awal pohon. Semua data masih berkumpul di sini sebelum dibagi.

Decision node

Di sini model bertanya sesuatu, misalnya apakah petal length lebih kecil dari batas tertentu.

Leaf node

Ini adalah ujung cabang. Di sinilah keputusan akhirnya muncul, misalnya kelas Setosa.

Demo alur keputusan Decision Tree

Demo ini menunjukkan bagaimana satu data bisa melewati cabang demi cabang sampai akhirnya masuk ke satu kelas.

Criterion

Ini adalah cara model menilai split terbaik. Secara sederhana, model mencari pembagian yang membuat kelas menjadi lebih rapi.

min_samples_leaf

Jumlah minimum data yang harus tersisa di leaf. Ini membantu agar daun tidak terlalu kecil.

max_features

Berapa banyak fitur yang dipertimbangkan saat model mencari split terbaik.

Splitter

Menentukan cara pencarian split, apakah memilih split terbaik atau strategi lainnya.

Kelebihan Decision Tree

  • Mudah dipahami karena bentuknya seperti aturan if-else.
  • Mudah divisualisasikan dan cocok untuk penjelasan dasar machine learning.
  • Tidak harus selalu melakukan standardisasi data.

Kekurangan Decision Tree

  • Mudah overfitting jika pohon terlalu dalam.
  • Perubahan kecil pada data bisa mengubah struktur pohon cukup besar.
  • Sering kurang stabil dibanding model ensemble.

Random Forest

Random Forest adalah algoritma ensemble learning yang menggabungkan banyak Decision Tree. Setiap pohon dilatih dengan sebagian data dan sebagian fitur secara acak. Setelah itu, semua pohon memberikan prediksi, lalu hasil akhir dipilih berdasarkan voting.

Intinya, kalau satu Decision Tree seperti satu orang yang memberi pendapat, maka Random Forest seperti meminta pendapat dari banyak orang, lalu mengambil jawaban yang paling banyak disetujui.

Kenapa dibuat banyak tree?

Karena satu Decision Tree saja bisa terlalu mengikuti data latihan. Dengan menggabungkan banyak tree, hasil akhir biasanya lebih stabil dan tidak terlalu mudah overfitting.

Intuisi voting

Kalau 5 tree memberi jawaban dan 3 di antaranya memilih kelas yang sama, maka kelas itulah yang menjadi prediksi akhir Random Forest.

Demo voting Random Forest

Silakan ubah tingkat kesulitan data dan jumlah pohon untuk melihat bagaimana hasil voting berubah.

n_estimators

Jumlah pohon yang digunakan. Semakin banyak, biasanya hasil lebih stabil, tetapi komputasi juga bertambah.

max_depth

Mengatur kedalaman tiap pohon. Tetap penting agar pohon tidak terlalu berlebihan.

min_samples_split

Jumlah minimum data agar node pada tiap tree boleh dipecah lagi.

min_samples_leaf

Jumlah minimum data di daun. Membantu hasil tree tidak terlalu ekstrem.

max_features

Jumlah fitur acak yang dilihat setiap tree saat mencari split terbaik.

bootstrap

Menentukan apakah data untuk tiap tree diambil secara sampling dengan pengembalian. Ini membuat tiap tree belajar dari versi data yang sedikit berbeda.

Kelebihan Random Forest

  • Biasanya lebih stabil daripada satu Decision Tree.
  • Lebih tahan terhadap overfitting dibanding pohon tunggal.
  • Sering memberi performa yang baik tanpa terlalu banyak tuning rumit.

Kekurangan Random Forest

  • Lebih berat dibanding satu pohon keputusan.
  • Lebih sulit dijelaskan detailnya daripada Decision Tree tunggal.
  • Kalau pohonnya terlalu banyak, model bisa makin lambat.

4. Cara membaca hasil evaluasi model

Setelah model dijalankan, mahasiswa akan melihat angka-angka seperti accuracy, precision, recall, F1-score, dan confusion matrix. Bagian ini membantu mereka memahami arti angka tersebut dengan bahasa sederhana.

Istilah evaluasi

  • Accuracy = persentase seluruh prediksi yang benar.
  • Precision = dari semua data yang diprediksi sebagai suatu kelas, berapa yang benar-benar tepat.
  • Recall = dari semua data yang memang milik suatu kelas, berapa yang berhasil ditemukan model.
  • F1-score = ukuran keseimbangan antara precision dan recall.
Penjelasan cepat yang mudah: accuracy melihat benar secara keseluruhan, precision melihat ketepatan saat model menebak, recall melihat kelengkapan saat model mencari suatu kelas.

Ringkasan penutup untuk disampaikan ke mahasiswa

SVM fokus mencari batas pemisah terbaik, Decision Tree fokus pada aturan keputusan bertahap, dan Random Forest fokus pada penggabungan banyak keputusan agar hasilnya lebih stabil. Ketiganya sama-sama bisa dipakai untuk klasifikasi, tetapi cara berpikirnya berbeda. Kalau mahasiswa sudah paham intuisi ini, maka saat masuk ke kode dan hasil evaluasi, mereka akan jauh lebih mudah mengikuti praktikum.
